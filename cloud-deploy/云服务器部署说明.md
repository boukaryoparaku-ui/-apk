## 🎯 两种部署方式

### Docker 部署（推荐）✅

**优势**：
- 环境隔离，易于迁移
- 一条命令启动
- 适合大多数云服务器

**要求**：
- 1核 1GB 内存
- Ubuntu 22.04+ / Debian 12+

**文档**：`docs/快速开始.md` 或 `docs/Docker部署.md`

### systemd 裸机部署

**优势**：
- 资源占用更低
- 适合小内存服务器

**要求**：
- 1核 512MB 内存
- 需要手动安装 Node.js 24

**文档**：`systemd/deploy-systemd.md`

### Docker 部署

```bash
# 进入项目目录
cd /opt/fashion-inventory

# 配置环境变量
cp cloud-deploy/.env.production.example .env
nano .env  # 修改密钥和密码

# 启动服务
docker compose -f cloud-deploy/docker-compose.prod.yml up -d --build

# 查看日志
docker compose -f cloud-deploy/docker-compose.prod.yml logs -f

# 健康检查
curl http://127.0.0.1:3001/api/healthz
```

### systemd 部署

```bash
# 进入项目目录
cd /opt/fashion-inventory

# 安装依赖
npm ci --omit=dev

# 配置环境变量
cp .env.example .env
nano .env  # 修改密钥和密码

# 初始化
npm run prod:init

# 配置服务
sudo cp cloud-deploy/systemd/fashion-inventory.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable fashion-inventory
sudo systemctl start fashion-inventory

# 查看日志
journalctl -u fashion-inventory -f
```

## 🔐 安全提醒

部署后**必须**完成以下安全措施：

### 应用安全
- ✅ 修改 `.env` 中的 `SESSION_SECRET`（随机密钥）
- ✅ 修改 `.env` 中的 `AI_CONFIG_SECRET`（随机密钥）
- ✅ 修改 `.env` 中的 `ADMIN_PASSWORD`（强密码）
- ✅ 配置 HTTPS（公网访问必需）

### 系统安全
- ✅ 配置防火墙（UFW / firewalld）
- ✅ 禁用 SSH root 登录
- ✅ 安装 fail2ban（防暴力破解）

### 数据安全
- ✅ 配置每日自动备份
- ✅ 备份同步到服务器外（对象存储/NAS）
- ✅ 测试数据恢复流程

**详细指南**：`cloud-deploy/docs/安全加固.md`

## 🌐 配置 HTTPS

### 使用 Caddy（推荐 - 自动 HTTPS）

```bash
# 安装 Caddy
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' | sudo gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' | sudo tee /etc/apt/sources.list.d/caddy-stable.list
sudo apt update && sudo apt install caddy

# 使用配置模板
sudo cp cloud-deploy/caddy.conf /etc/caddy/Caddyfile
sudo nano /etc/caddy/Caddyfile  # 修改域名
sudo systemctl reload caddy
```

### 使用 Nginx + Certbot

```bash
# 安装
sudo apt install -y nginx certbot python3-certbot-nginx

# 使用配置模板
sudo cp cloud-deploy/nginx.conf /etc/nginx/sites-available/fashion-inventory
sudo ln -s /etc/nginx/sites-available/fashion-inventory /etc/nginx/sites-enabled/
sudo nano /etc/nginx/sites-available/fashion-inventory  # 修改域名
sudo nginx -t && sudo systemctl restart nginx

# 申请证书
sudo certbot --nginx -d your-domain.com
```

## 💾 配置自动备份

```bash
# Docker 部署
crontab -e
# 添加：每天凌晨2点备份
0 2 * * * /opt/fashion-inventory/cloud-deploy/scripts/backup.sh docker >> /var/log/fashion-backup.log 2>&1

# systemd 部署
sudo crontab -e -u fashion
# 添加：每天凌晨2点备份
0 2 * * * /opt/fashion-inventory/cloud-deploy/scripts/backup.sh systemd >> /var/log/fashion-backup.log 2>&1
```

备份文件位置：`/opt/fashion-inventory/backups/`

**重要**：务必将备份同步到服务器外！
