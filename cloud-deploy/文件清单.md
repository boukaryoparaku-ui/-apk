# 云服务器部署文件清单

本文档列出 `cloud-deploy/` 目录中的所有文件及其用途。

## 📁 文件结构

```
cloud-deploy/
├── README.md                        # 主部署文档
├── docker-compose.prod.yml          # Docker Compose 生产配置
├── Dockerfile                       # Docker 镜像构建文件
├── docker-entrypoint.sh             # 容器启动脚本
├── .env.production.example          # 生产环境变量模板
├── nginx.conf                       # Nginx 反向代理配置示例
├── caddy.conf                       # Caddy 反向代理配置示例
│
├── scripts/                         # 维护脚本
│   ├── backup.sh                   # 数据库备份脚本（支持 Docker 和 systemd）
│   ├── deploy.sh                   # 一键部署脚本（Docker）
│   └── update.sh                   # 更新脚本（拉取代码并重启）
│
├── systemd/                         # systemd 裸机部署
│   ├── fashion-inventory.service   # systemd 服务单元文件
│   ├── backup.sh                   # 裸机部署备份脚本
│   └── deploy-systemd.md           # systemd 部署详细文档
│
└── docs/                            # 详细文档
    ├── 快速开始.md                  # 5分钟快速部署指南
    ├── Docker部署.md               # Docker 部署完整文档
    └── 安全加固.md                  # 安全配置指南
```

## 📄 文件说明

### 核心文件

#### README.md
- **用途**: 部署方案主文档
- **内容**:
  - 方案选择（Docker vs systemd）
  - 快速部署步骤
  - HTTPS 配置
  - 备份恢复
  - 故障排查
  - 安全检查清单

#### docker-compose.prod.yml
- **用途**: Docker Compose 生产环境配置
- **特性**:
  - 环境变量从 `.env` 读取
  - 数据持久化卷配置
  - 健康检查
  - 资源限制
  - 日志轮转

#### Dockerfile
- **用途**: 构建应用 Docker 镜像
- **特性**:
  - 多阶段构建（deps → build → runtime）
  - 基于 Node.js 24 Alpine
  - 优化镜像大小（~150MB）
  - 生产优化

#### docker-entrypoint.sh
- **用途**: 容器启动时执行
- **功能**:
  - 创建必要目录
  - 初始化数据库（`prisma db push`）
  - 启动应用

#### .env.production.example
- **用途**: 生产环境变量模板
- **必须修改**:
  - `SESSION_SECRET`: 会话密钥
  - `AI_CONFIG_SECRET`: AI Key 加密密钥
  - `ADMIN_PASSWORD`: 管理员密码

### 反向代理配置

#### nginx.conf
- **用途**: Nginx 反向代理配置示例
- **特性**:
  - 反向代理到应用
  - 传递真实客户端 IP
  - 超时配置
  - 日志配置
  - 健康检查端点
- **配置位置**: `/etc/nginx/sites-available/`

#### caddy.conf
- **用途**: Caddy 反向代理配置示例
- **特性**:
  - 自动 HTTPS（Let's Encrypt）
  - 自动证书续期
  - 简洁配置
  - 健康检查
- **配置位置**: `/etc/caddy/Caddyfile`

### 维护脚本

#### scripts/backup.sh
- **用途**: 数据库备份脚本
- **支持**:
  - Docker 部署（从 volume 备份）
  - systemd 部署（直接备份文件）
- **功能**:
  - 自动压缩备份
  - 清理旧备份（默认保留14天）
  - 显示备份大小和数量
- **用法**: `bash backup.sh docker` 或 `bash backup.sh systemd`

#### scripts/deploy.sh
- **用途**: 一键部署脚本（Docker 方式）
- **功能**:
  - 检查 Docker 是否安装
  - 创建项目目录
  - 复制文件
  - 生成随机密钥
  - 启动服务
  - 健康检查
- **用法**: `sudo bash deploy.sh`

#### scripts/update.sh
- **用途**: 更新应用脚本
- **功能**:
  - 拉取最新代码（git pull）
  - 重新构建 Docker 镜像
  - 重启服务
  - 健康检查
- **用法**: `bash update.sh`

### systemd 部署

#### systemd/fashion-inventory.service
- **用途**: systemd 服务单元文件
- **特性**:
  - 自动重启
  - 内存限制
  - 安全加固（NoNewPrivileges, PrivateTmp）
  - 以专用用户运行
- **配置位置**: `/etc/systemd/system/`

#### systemd/backup.sh
- **用途**: 裸机部署备份脚本
- **同**: `scripts/backup.sh` 的软链接或副本

#### systemd/deploy-systemd.md
- **用途**: systemd 裸机部署完整文档
- **内容**:
  - Node.js 安装
  - 项目配置
  - systemd 服务配置
  - 反向代理配置
  - 性能优化
  - 故障排查

## 🚀 使用流程

### Docker 部署流程

1. 阅读 `README.md` 或 `docs/快速开始.md`
2. 上传项目到服务器
3. 复制 `.env.production.example` 为 `.env` 并修改
4. 运行 `docker compose -f docker-compose.prod.yml up -d --build`
5. 配置反向代理（`nginx.conf` 或 `caddy.conf`）
6. 设置自动备份（`scripts/backup.sh`）

### systemd 部署流程

1. 阅读 `systemd/deploy-systemd.md`
2. 安装 Node.js
3. 上传项目并安装依赖
4. 配置 `.env`
5. 运行 `npm run prod:init`
6. 配置 systemd 服务（`systemd/fashion-inventory.service`）
7. 配置反向代理
8. 设置自动备份

## 📝 自定义和扩展

### 修改端口

编辑 `.env`:
```env
HOST_PORT=8080  # Docker 部署
PORT=8080       # systemd 部署
```

### 修改内存限制

编辑 `docker-compose.prod.yml`:
```yaml
deploy:
  resources:
    limits:
      memory: 1G  # 改成所需大小
```

或编辑 `systemd/fashion-inventory.service`:
```ini
MemoryMax=768M  # 改成所需大小
```

### 添加自定义域名

编辑 `nginx.conf` 或 `caddy.conf`，添加新的 server/域名块。
