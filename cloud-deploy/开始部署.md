# 云服务器部署方案 - 快速导航

欢迎使用服装供销存系统云服务器部署方案！

## 🎯 选择你的部署方式

### 🐳 Docker 部署（推荐）
**适合**：大多数场景，尤其是首次部署

- ✅ 环境隔离、迁移方便
- ✅ 一条命令启动
- ⚠️ 需要 1GB+ 内存

📖 [5分钟快速开始](docs/快速开始.md) → [详细文档](docs/Docker部署.md)

### 🖥️ systemd 裸机部署
**适合**：小内存服务器（512MB）

- ✅ 资源占用低
- ⚠️ 需要手动安装 Node.js
- ⚠️ 配置相对复杂

📖 [systemd 部署详细文档](systemd/deploy-systemd.md)

## 🛠️ 核心文件

| 文件 | 用途 |
|------|------|
| `docker-compose.prod.yml` | Docker 生产配置 |
| `.env.production.example` | 环境变量模板 |
| `nginx.conf` / `caddy.conf` | 反向代理配置示例 |
| `scripts/backup.sh` | 数据库备份脚本 |
| `scripts/deploy.sh` | 一键部署脚本 |
| `systemd/fashion-inventory.service` | systemd 服务配置 |

## ⚡ 快速命令

### Docker 部署
```bash
# 启动
docker compose -f cloud-deploy/docker-compose.prod.yml up -d --build

# 查看日志
docker compose -f cloud-deploy/docker-compose.prod.yml logs -f

# 重启
docker compose -f cloud-deploy/docker-compose.prod.yml restart

# 备份
bash cloud-deploy/scripts/backup.sh docker
```

### systemd 部署
```bash
# 启动
sudo systemctl start fashion-inventory

# 查看日志
journalctl -u fashion-inventory -f

# 重启
sudo systemctl restart fashion-inventory

# 备份
bash cloud-deploy/scripts/backup.sh systemd
```

## 🔐 安全提醒

部署后**必须**完成：

1. ✅ 修改 `.env` 中的所有密钥和密码
2. ✅ 配置 HTTPS（公网访问必需）
3. ✅ 设置防火墙
4. ✅ 配置自动备份

详见：[docs/安全加固.md](docs/安全加固.md)

## 📦 部署流程概览

```
1. 选择部署方式 (Docker 或 systemd)
   ↓
2. 准备服务器（安装 Docker 或 Node.js）
   ↓
3. 上传项目文件
   ↓
4. 配置环境变量（.env）
   ↓
5. 启动服务
   ↓
6. 配置 HTTPS（可选但推荐）
   ↓
7. 设置自动备份
   ↓
8. 完成安全加固
```
