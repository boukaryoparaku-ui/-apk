# 安全加固指南

本文档提供云服务器部署后的安全加固建议。

## 🔐 核心安全措施

### 1. 修改默认密码

部署完成后**立即**修改管理员密码。

```bash
# 编辑环境变量
nano /opt/fashion-inventory/.env

# 修改 ADMIN_PASSWORD 为强密码：
# - 至少 12 位
# - 包含大小写字母、数字、特殊字符
ADMIN_PASSWORD=YourStrongP@ssw0rd!2024

# Docker 部署重启
docker compose -f /opt/fashion-inventory/cloud-deploy/docker-compose.prod.yml restart

# systemd 部署重启
sudo systemctl restart fashion-inventory
```

登录后在「设置 → 账号管理」中修改密码。

### 2. 保护敏感密钥

#### SESSION_SECRET 和 AI_CONFIG_SECRET

这两个密钥非常重要：

- **SESSION_SECRET**: 保护用户登录会话
- **AI_CONFIG_SECRET**: 加密数据库中的 AI API Key

**安全要求**：
- 使用随机生成的 32 字节十六进制字符串
- 生产环境**绝不能**使用默认值
- `AI_CONFIG_SECRET` 一旦设定**不要随意更改**（否则已加密的 API Key 无法解密）

```bash
# 生成强密钥
openssl rand -hex 32
```

#### 保护 .env 文件

```bash
# 限制文件权限
chmod 600 /opt/fashion-inventory/.env
sudo chown fashion:fashion /opt/fashion-inventory/.env  # systemd 部署

# 确保 .env 不会被提交到 git
grep -q ".env" /opt/fashion-inventory/.gitignore || echo ".env" >> /opt/fashion-inventory/.gitignore
```

### 3. 配置 HTTPS（必需）

**公网访问时必须启用 HTTPS**，否则密码和数据明文传输。

#### 使用 Caddy（推荐 - 自动 HTTPS）

```bash
# 安装 Caddy
sudo apt install -y debian-keyring debian-archive-keyring apt-transport-https
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' | \
  sudo gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' | \
  sudo tee /etc/apt/sources.list.d/caddy-stable.list
sudo apt update
sudo apt install caddy

# 配置反向代理
sudo tee /etc/caddy/Caddyfile > /dev/null <<EOF
your-domain.com {
    reverse_proxy 127.0.0.1:3001

    # 安全头
    header {
        Strict-Transport-Security "max-age=31536000; includeSubDomains"
        X-Content-Type-Options "nosniff"
        X-Frame-Options "DENY"
        X-XSS-Protection "1; mode=block"
        Referrer-Policy "strict-origin-when-cross-origin"
    }
}
EOF

# 重启 Caddy
sudo systemctl reload caddy
```

Caddy 会自动申请和续期 Let's Encrypt 证书。

#### 使用 Nginx + Certbot

```bash
# 安装
sudo apt install -y nginx certbot python3-certbot-nginx

# 配置反向代理
sudo cp /opt/fashion-inventory/cloud-deploy/nginx.conf /etc/nginx/sites-available/fashion-inventory
sudo ln -s /etc/nginx/sites-available/fashion-inventory /etc/nginx/sites-enabled/
sudo nano /etc/nginx/sites-available/fashion-inventory
# 修改 your-domain.com

# 测试配置
sudo nginx -t
sudo systemctl restart nginx

# 申请 SSL 证书
sudo certbot --nginx -d your-domain.com

# 设置自动续期
sudo systemctl enable certbot.timer
```

### 4. 配置防火墙

#### 使用 UFW（Ubuntu/Debian）

```bash
# 启用防火墙
sudo ufw default deny incoming
sudo ufw default allow outgoing

# 允许 SSH（避免锁定自己）
sudo ufw allow ssh

# 允许 HTTP/HTTPS
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# 如果直接访问应用端口（不推荐）
# sudo ufw allow 3001/tcp

# 启用
sudo ufw enable
sudo ufw status
```

#### 使用 firewalld（CentOS/RHEL）

```bash
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --permanent --add-service=ssh
sudo firewall-cmd --reload
```

### 5. 限制应用端口访问

如果使用反向代理，应用端口应该只监听本地。

#### Docker 部署

编辑 `.env`：

```env
# 只允许本机访问
HOST_PORT=127.0.0.1:3001
```

重启服务：

```bash
docker compose -f /opt/fashion-inventory/cloud-deploy/docker-compose.prod.yml up -d
```

#### systemd 部署

编辑 `.env`：

```env
# 绑定到本地回环地址
HOST=127.0.0.1
PORT=3000
```

## 🛡️ 系统级安全

### 6. 禁用 Root SSH 登录

```bash
# 编辑 SSH 配置
sudo nano /etc/ssh/sshd_config

# 修改以下配置
PermitRootLogin no
PasswordAuthentication no  # 强制使用密钥登录
PubkeyAuthentication yes

# 重启 SSH
sudo systemctl restart sshd
```

### 7. 配置 Fail2Ban（防暴力破解）

```bash
# 安装
sudo apt install -y fail2ban

# 配置
sudo tee /etc/fail2ban/jail.local > /dev/null <<EOF
[sshd]
enabled = true
port = ssh
filter = sshd
logpath = /var/log/auth.log
maxretry = 5
bantime = 3600
EOF

# 启动
sudo systemctl enable fail2ban
sudo systemctl start fail2ban

# 查看状态
sudo fail2ban-client status sshd
```

### 8. 启用自动安全更新

```bash
# Ubuntu/Debian
sudo apt install -y unattended-upgrades
sudo dpkg-reconfigure -plow unattended-upgrades

# 配置只更新安全补丁
sudo nano /etc/apt/apt.conf.d/50unattended-upgrades
# 确保启用：
# Unattended-Upgrade::Automatic-Reboot "false";
```

### 9. 配置系统日志监控

```bash
# 安装 logwatch
sudo apt install -y logwatch

# 配置每日邮件摘要（可选）
sudo nano /etc/cron.daily/00logwatch
```

### 10. 限制文件权限

```bash
# 项目目录权限
sudo chown -R fashion:fashion /opt/fashion-inventory
sudo chmod -R 755 /opt/fashion-inventory

# 敏感文件权限
sudo chmod 600 /opt/fashion-inventory/.env
sudo chmod 600 /opt/fashion-inventory/prisma/data.db
```

## 💾 数据安全

### 11. 配置加密备份

```bash
# 安装 gpg
sudo apt install -y gnupg

# 生成密钥（如果没有）
gpg --gen-key

# 修改备份脚本，添加加密
nano /opt/fashion-inventory/cloud-deploy/scripts/backup.sh

# 在压缩后添加：
# gpg --encrypt --recipient your-email@example.com "${BACKUP_FILE}.gz"

```

### 12. 备份到远程存储

**重要**：备份必须同步到服务器外！

#### 使用 rclone（推荐）

```bash
# 安装 rclone
curl https://rclone.org/install.sh | sudo bash

# 配置云存储
rclone config
# 按提示配置对象存储/网盘

# 添加到备份脚本
# rclone sync /opt/fashion-inventory/backups/ remote:fashion-backups/
```

#### 使用 rsync

```bash
# 同步到另一台服务器
rsync -avz --delete \
  /opt/fashion-inventory/backups/ \
  user@backup-server:/backups/fashion-inventory/
```

#### 定时同步

```bash
crontab -e
# 每天凌晨3点同步备份
0 3 * * * rclone sync /opt/fashion-inventory/backups/ remote:fashion-backups/ >> /var/log/backup-sync.log 2>&1
```

### 13. 数据库加密（高级）

SQLite 支持加密，但需要编译扩展。简单方案：

```bash
# 加密备份文件
gzip /path/to/data.db
openssl enc -aes-256-cbc -salt -in data.db.gz -out data.db.gz.enc
# 保存密码到安全位置
```

## ⚠️ 应急响应

### 发现安全问题

1. **立即断网隔离**（如果严重）
   ```bash
   sudo systemctl stop fashion-inventory
   sudo ufw deny 80
   sudo ufw deny 443
   ```

2. **查看访问日志**
   ```bash
   # Docker
   docker compose -f /opt/fashion-inventory/cloud-deploy/docker-compose.prod.yml logs | grep -i "error\|failed\|unauthorized"

   # systemd
   journalctl -u fashion-inventory --since "1 hour ago" | grep -i "error\|failed\|unauthorized"
   ```

3. **修改所有密码**
   - 管理员密码
   - SESSION_SECRET
   - AI_CONFIG_SECRET（需要重新配置 AI）
   - 服务器 root 密码

4. **恢复备份**（如果数据被篡改）

### 密钥泄露处理

如果 `SESSION_SECRET` 泄露：

```bash
# 生成新密钥
NEW_SECRET=$(openssl rand -hex 32)

# 更新 .env
sed -i "s/SESSION_SECRET=.*/SESSION_SECRET=$NEW_SECRET/" .env

# 重启（会踢出所有登录用户）
docker compose -f /opt/fashion-inventory/cloud-deploy/docker-compose.prod.yml restart
```

如果 `AI_CONFIG_SECRET` 泄露：

⚠️ **慎重**：更改后已保存的 AI Key 无法解密。

1. 先备份数据库
2. 更新密钥
3. 重新配置 AI API Key
