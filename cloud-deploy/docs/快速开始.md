# 快速开始 - 5分钟部署指南

本文档帮助你在 5 分钟内完成最基础的部署。

## 前提条件

- ✅ 一台云服务器（1核1G，Ubuntu 22.04）
- ✅ 已获取服务器 SSH 登录权限
- ✅ 服务器能访问互联网

## Docker 部署（推荐）

### 1. 连接服务器

```bash
ssh user@your-server-ip
```

### 2. 安装 Docker

```bash
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
```

退出并重新登录。

### 3. 上传项目

```bash
# 方法1：Git（推荐）
cd /opt
sudo git clone https://github.com/your-repo/project.git fashion-inventory
sudo chown -R $USER:$USER fashion-inventory

# 方法2：从本地上传
# scp -r /local/path user@server:/opt/fashion-inventory
```

### 4. 配置环境

```bash
cd /opt/fashion-inventory
cp cloud-deploy/.env.production.example .env

# 生成随机密钥
openssl rand -hex 32  # 复制输出
nano .env  # 粘贴到 SESSION_SECRET=后面

openssl rand -hex 32  # 复制输出
nano .env  # 粘贴到 AI_CONFIG_SECRET=后面

# 修改 ADMIN_PASSWORD 为你的密码
```

### 5. 启动服务

```bash
docker compose -f cloud-deploy/docker-compose.prod.yml up -d --build
```

### 6. 验证

```bash
curl http://127.0.0.1:3001/api/healthz
# 返回 {"ok":true,"db":"up"} 表示成功
```

浏览器访问：`http://服务器IP:3001`

## systemd 裸机部署（小内存服务器）

### 1. 安装 Node.js

```bash
curl -fsSL https://deb.nodesource.com/setup_24.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### 2. 上传项目

```bash
sudo mkdir -p /opt/fashion-inventory
sudo chown -R $USER:$USER /opt/fashion-inventory
cd /opt/fashion-inventory
# 上传或 git clone 项目文件
```

### 3. 安装依赖

```bash
npm ci --omit=dev
```

### 4. 配置环境

```bash
cp .env.example .env
nano .env
# 修改 SESSION_SECRET, AI_CONFIG_SECRET, ADMIN_PASSWORD
```

### 5. 初始化

```bash
npm run prod:init
```

### 6. 配置服务

```bash
sudo cp cloud-deploy/systemd/fashion-inventory.service /etc/systemd/system/
sudo useradd --system --no-create-home --shell /usr/sbin/nologin fashion
sudo chown -R fashion:fashion /opt/fashion-inventory
sudo systemctl daemon-reload
sudo systemctl enable fashion-inventory
sudo systemctl start fashion-inventory
```

### 7. 验证

```bash
curl http://127.0.0.1:3000/api/healthz
```

浏览器访问：`http://服务器IP:3000`

## 配置 HTTPS（可选但推荐）

### 使用 Caddy（最简单）

```bash
# 安装 Caddy
sudo apt install -y debian-keyring debian-archive-keyring apt-transport-https
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' | sudo gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' | sudo tee /etc/apt/sources.list.d/caddy-stable.list
sudo apt update && sudo apt install caddy

# 配置
sudo cp cloud-deploy/caddy.conf /etc/caddy/Caddyfile
sudo nano /etc/caddy/Caddyfile
# 修改 your-domain.com 为你的域名

# 重启
sudo systemctl reload caddy
```

访问：`https://your-domain.com`（自动 HTTPS）

## 设置自动备份

```bash
# 复制脚本
chmod +x cloud-deploy/scripts/backup.sh

# 添加定时任务
crontab -e
# 添加这一行（每天凌晨2点备份）：
0 2 * * * /opt/fashion-inventory/cloud-deploy/scripts/backup.sh docker >> /var/log/fashion-backup.log 2>&1
```

## 常见问题

### 端口无法访问

```bash
# 开放防火墙
sudo ufw allow 3001  # Docker 部署
sudo ufw allow 3000  # systemd 部署
sudo ufw allow 80
sudo ufw allow 443
```

### 服务无法启动

```bash
# Docker
docker compose -f cloud-deploy/docker-compose.prod.yml logs -f

# systemd
journalctl -u fashion-inventory -f
```

### 内存不足

```bash
# 添加 swap
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```
